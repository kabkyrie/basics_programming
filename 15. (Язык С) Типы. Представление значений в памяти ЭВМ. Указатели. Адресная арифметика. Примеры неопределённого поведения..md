
***Типы данных*** - дополнительная информация об объекте, которая
- Определяет (или ограничивает) его размер в памяти
- Описывает структуру (внутренне устройство)
- Определяет контекст, в котором он может использоваться
	Определяет набор допустимых или недопустимых операций, которые могут быть совершены над объектом

По сути тип данных - набор ограничений, для корректной работы с ним.

---
### Простые типы
В Си для многих базовых типов размер не определен, лишь введены ограничения.

- `char` - 1 байт
- `short` - 2 байта
- `int` - 4 байта
- `long` - 4 - 8 байтов в зависимости от ос
- `long long` - 8 байт

- `viod` - неполный тип (используется для функций, которые ничего не возвращают или как указатель)

Типы с явными размерами из `stdint.h`: `int8_t`, `uint16_t`, `int32_t`

Беззнаковые (unsingned) - в памяти представляются прямой двоичной записью
Знаковые (signed) - в памяти представляются в дополнительном коде

### Вещественные типы

- `float` - 4 байта
- `double` - 8 байт
- `long double` - 10, 12, 16 байт

В памяти представляются по стандарту IEEE-754
Подробнее в билете [[13. ​Представление данных в памяти ЭВМ представление вещественных с плавающей точкой, (NaN, Inf). Нюансы с fixed point. RISC-V soft-floatcalling convention.]]

### Указатели
***Указатель*** - переменная, которая хранит номер ячейки в памяти (адрес)
С точки зрения памяти, указатель - обычное беззнаковое целое число.
`int*`, `char*` ...

Размер указателя зависит только от архитектуры процессора
- 32-битная система - указатель 4 байта
- 64 битная система - указатель 8 байт

---
### Составные типы

##### **Массивы** (arrays) - "одинаковые подряд"
В памяти представлены как непрерывный блок
`int arr[3] = {1, 2, 3}`

##### **Структуры** (struct) - "разные одновременно". 
```C
struct my_struct {
	char x; // 1 байт
	// -- 3 байта мусора (padding) --
	int y; // 4 байта
};
```
В памяти может представляться с выравниванием (aligment) или без выравнивания.

Располагать элементы в структуре лучше от самого большого к самому маленькому

***Aligment*** - требование процессора, чтобы данные в памяти располагались по адресам, кратным их размеру. (Процессор вставляет между полями структуры пустые байты (padding), чтобы процессору было удобнее и эффективнее читать) ^54a655

Размер всей структуры выравнивается по самому большому типу внутри неё

Можно отключить выравнивание - `#pragma pack (1)` (для сильной оптимизации памяти)

##### **Объединения** (union) - "разные взаимоисключающие"
```C
union Data {
    int i;
    float f;
    char str[20];
};
```
Хранит разные типы данных в одной и той же области памяти, она каждый раз перезаписывается. Размер равен размеру его самого большого элемента.

### Адресная арифметика

***Адресная арифметика*** - способ вычисления адреса при помощи арифметических действий над указателями.

Операторы:
- `&` - взятие адреса
- `*` - разыменование (взятие значения по адресу)
- `->` - доступ к полю структуры через указатель

Основные правила:
1) Шаг арифметики
	- `p + 1` - увеличивает адрес не на 1 байт, а на `sizeof(*p)` байт
2) Допустимые операции
	- `ptr1 - ptr2` - расстояние между двумя указателями (в штуках элементов, а не в байтах)
	- `>` `<` `==` - какой указатель в памяти дальше.

---
### Примеры неопределенного поведения
1) Выход за границу массива (может привести к `segmentation fault`)
2) Разыменование\неинициализированного нулевого указателя
3) Целочисленное деление на ноль
4) Переполнение целого со знаком
5) Выход за пределы времени жизни объекта (Возврат указателя на объект в стеке)
