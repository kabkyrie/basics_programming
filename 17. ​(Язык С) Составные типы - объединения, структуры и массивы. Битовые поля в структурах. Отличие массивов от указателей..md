
***Объединения*** (union) - один из способов описывать типы.
В памяти одновременно может храниться один из нескольких типов.
Размер буфера для хранения - наибольший из размеров элементов.
Запись и чтение происходит как обращение к полю соответствующего типа.
Компилятор не проверяет, что тип, по которому производилась последняя запись совпадает с типом, по которому значение читается, то есть можно интерпретировать данные неправильно.

***Структуры*** (struct) - один из способов описывать типы.
В памяти одновременно хранится фиксированное количество элементов разных типов
Размер буфера для хранения - сумма размеров элементов + выравнивание.

***Массивы*** (array) - именованная область памяти фиксированного размера, в которой элементы одного типа хранятся последовательно в памяти.
Есть массивы, размер которых известен на этапе компиляции - они объявляются с помощью оператора `[]`, для них `sizeof()` возвращает длину умноженную на размер типа.
Массивы можно организовать руками, просто выделив буфер достаточного размера любого типа (например через `malloc()`) и вернув на него указатель, тогда длина будет известна (`sizeof` возвращает размер указателя)

**malloc()** - функция, которая используется для динамического выделения памяти во время работы программы. Находится в библиотеке `stdlib.h`. Она принимает количество байт, которое нужно выделить и возвращает указатель на начало выделенного блока. В конце нужно отчистить `free(arr)`

### Размещение в памяти, выравнивание (alignment)
Подробнее в билете [[15. (Язык С) Типы. Представление значений в памяти ЭВМ. Указатели. Адресная арифметика. Примеры неопределённого поведения.#^54a655]]

Массивы выравниваются также как и хранимый в них тип
```C
int main()
{
	char c;
	// +3 байта для выравнимвания int[]
	int i[10];
{
```

Размер `union` - максимум из размеров его компонентов, выравнивание перед ним равно выравниванию максимального компонента

Размер `struct` - сумма размеров его компонентов + выравнивание.

Директива `#prgma pack(n)` ограничивает выравнивание сверху до `n`
```C
#pragma pack(push, 1)

struct Foo
{
	int i;
	char c;
	// +0 байт
}
#pragma pack(pop)
```

### Битовые поля в структурах
Позволяет переопределить размер поля в структуре
```C
struct Flags {
    unsigned int ready : 1;   // 1 бит
    unsigned int error : 1;   // 1 бит
    unsigned int mode  : 2;   // 2 бита
    unsigned int code  : 4;   // 4 бита
};
```
Размер этой структуры будет 4 байта (размер `int`)
Работает только для целочисленных типов.

### Отличие массивов от указателей

Отличие в поведении `sizeof`
- по переменной типа `type arr[]` можно узнать размер массива (число элементов умноженное на размер массива)
- по переменной типа `type *ptr` можно узнать только размер указателя

Отличие мутабельности (свойство объекта, позволяющее изменять его после создания)
- переменная типа `type arr[]` немутабельна
- переменной типа `type *ptr` мутабельны
