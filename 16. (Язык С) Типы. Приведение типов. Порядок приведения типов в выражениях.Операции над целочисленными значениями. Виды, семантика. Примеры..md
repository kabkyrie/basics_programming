
***Приведение типа*** (cast) - преобразование одного типа к другому
- явное - программист пишет `(type)expr`, чтобы привести `expr` к типу `type`
- неявное - компилятор сам выполняет преобразование типа выражения

Неявное выполняется при:
- арифметических операциях
- возврате значения из функции
- присваивание (если выражение слева от `=` имеет не тот тип, который имеет переменная)

При приведении типов может:
- теряться точность (при `double` -> `float`)
- теряться дробная часть (при `float` -> `int`)
- происходить арифметическое переполнение (при `long` -> `int`)

### Порядок приведения типов в выражениях

***Usual Arithmetic Conversions*** - касты, которые автоматически выполняются в выражениях в Си.
Компилятор всегда старается привести меньший тип к большему (расширение), чтобы избежать переполнения.

1) **Integer Promotion** - если тип вкладывается в `int`, то перед арифметической операцией он приводится к `int`
	Часто выполняется для: `short`, `char`, `bool`, `enum`

2) **Иерархия типов** - если после 1-ого этапа типы все еще разные, включается правило приведения к старшему типу.
	`int` -> `unsigned int` -> `long` -> `unsigned long` -> `long long` ->
	-> `unsigned long long` -> `float` -> `double` -> `long double` 

3) Коварный случай - если встречаются знаковое (signed) и беззнаковое (unsigned) числа одинакового типа и размера (`int` и `unsigned int`), то знаковое число превращается в беззнаковое. Это приводит к некоторым ошибкам.
```C
int a = -1;
unsigned int b = 10;
if (a < b) {
 // Мы ждем, что -1 < 10 (True)
 printf("Все ок\n");
 } else {
// Но программа зайдет СЮДА!
 printf("Ошибка!\n");
 }
```
Потому что $-1$ в битах это $1111...1111_2$
В `unsigned int` это 4 с лишним миллиарда.

### Операции над целочисленными значениями. Виды, семантика, примеры

1) Арифметические: `+` `-` `*` `/` `%`

2) Логические: `||`  `&&`  `!`  `<` `>` `<=` `>=` `==` `!=`

3) Побитовые операции:  `|`  `&`  `^` `~`  `>>` `<<`
	`|` - побитовое или
	`&` - побитовое и
	`^` - исключающее или (XOR) (разные биты дают 1, а одинаковые 0)
	`~` - инверсия
	`<<` `>>` - сдвиги на 1 бит влево и вправо ( $0101$ << $1$ -> $1010$ ) ( $0101$ >> $1$ -> $0010$ )

4) Операции над указателями:  `*` `&` `->`
	`*` - получить значение по адресу 
	`&` - получить адрес переменной
	 `->` - доступ к полям через указатель `(*struct_ptr).x = 5;` сокращается до 
		 `struct_ptr->x = 5;`
