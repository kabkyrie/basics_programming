***Файл*** - идентифицируемая область памяти, являющаяся базовой единицей работы с данными в ОС. (абстракция уровня ОС) В UNIX всё - файл.

---
### Классификация файлов и основные свойства

1) **Regular file**
	сюда относят текст, бинарный код, мультимедиа, то есть большинство пользовательских файлов
	**Префикс в `ls -l`:** `-` (черточка)

2) **Directory (директория)**
	Контейнер для других файлов
	На самом деле, для системы это тоже файл, внутри которого просто лежит список: "имя файла - где он лежит на диске"
	 **Префикс в `ls -l`:** `d`

3) **Symbolic link (символическая ссылка)**
	Это крошечный файл, внутри которого написан _путь_ (адрес) к другому файлу.
	 **Префикс в `ls -l`:** `l`

4) **Hard link**
	Является равносильным идентификатором для файла, на который указывает.
	Нельзя создать hard link на директорию или файл на другом диске/разделе.
	Место на накопителе, где хранятся метаданные и данные файла называют inode.
	Hard link указывает на inode, symbolic link — содержит путь
	`ln file name` - создает hard link с именем name на файл file

5) **Device files**
	Так как файл, железо компьютера тоже представлено файлами в папке `/dev`
	Они делятся на два типа
	- Block device - устройства хранения данных (префикс `d`)
	- Charecter device - устройства, которые передают данные потоком, по одному байтику (клавиатура / мышь / терминал)
	- Префикс `c`

6) **FIFO / Named Pipe (именованный канал)**
	Способ общения двух программ. Одна программа пишет в этот файл, а другая читает из него. Префикс `p`

7) Socket
	Продвинутая версия канала для общения программ. Часто используется для локального взаимодействия сервисов. Префикс `s`

---
### Текстовый файл
***Текстовый файл*** - файл, состоящий из последовательности строк
Строка - последовательность символов, которая заканчивается `\n`.
`\0` - EOF (конец файла)

---
### ASCII - терминал
***ASCII - терминал*** - программа, которая взаимодействует с компьютером, отправляя и получая только текстовые символы (байты).

1) Удаленная эхо-печать - каждый посланный удаленному хосту символ возвращается назад до того как будет отображен на экране пользователя
2) Полнодуплексный эффект - поток символов одновременно передается в обоих направлениях
3) Поддержка полноэкранных интерактивных приложений (загружает сеть)
4) Набор символов ASCII больше набора символов NVT (network viryual terminal - очень старый стандарт для виртуальных терминалов)
