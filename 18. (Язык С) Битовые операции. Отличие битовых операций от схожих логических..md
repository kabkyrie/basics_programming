### Битовые операции: область применения, примеры оптимизации решения задач через битовые операции.


#### Области применения
- При разработке драйверов устройств, например программ для модемов
- При декодировании информации от внешних устройств. В шифровании
- Максимально быстрые вычисления
- Более эффективное хранение данных

---
#### Примеры оптимизации решения задач через битовые операции

1) **Оптимизация по времени**
	Мы может умножать и делить на степень двойки очень быстро
```C
int x = 50; // Умножаем на 8 (2^3)
// Обычный способ: int y = x * 8;
int fast_mul = x << 3; // 50 (110010) << 3 = 400 (110010000)

// Делим на 4 (2^2)
// Обычный способ: int z = x / 4;
int fast_div = x >> 2; // 50 (110010) >> 2 = 12 (1100) -> 12.5 округлилось до 12
```

2) **По памяти** 
	Можем хранить сразу 8 flags в одной переменной (так делать не надо, наверное, но как пример сойдет).
	Заводим переменную типа char и с помощью битовых операций можем каждый битик использовать как отдельный флаг. Достаем вот так: ((big_flag >> i) & 1), где i - номер флага.
```C
unsigned char flags = 0; // 0000 0000

// Определим номера битов
#define ACTIVE  0  // 0-й бит
#define WEAPON  1  // 1-й бит
#define JUMP    2  // 2-й бит

// 1. Установить флаг (сделать активным)
// Сдвигаем 1 на нужную позицию и делаем OR
flags |= (1 << ACTIVE); // flags: 0000 0001

// 2. Добавить оружие
flags |= (1 << WEAPON); // flags: 0000 0011

// 3. Проверить, есть ли оружие
// Сдвигаем флаги вправо, чтобы нужный бит стал последним, и берем & 1
if ((flags >> WEAPON) & 1) {
    printf("Есть оружие!");
}
```

3) **Работа с IP-адресами**
	Допустим есть IP и его маска, хотим найти адрес сети

```C
unsigned int ip = 0xC0A80169;   // 192.168.1.105 (hex)
unsigned int mask = 0xFFFFFF00; // 255.255.255.0 (hex)

// Чтобы получить адрес сети, нужно просто сделать AND
unsigned int network = ip & mask; 
// C0 & FF -> C0
// A8 & FF -> A8
// 01 & FF -> 01
// 69 & 00 -> 00
// Результат: C0A80100 -> 192.168.1.0
```

4) **Little Endian <-> Big Endian**
	**Ситуация:** Вы читаете файл или пакет из сети. Там числа записаны "старшим байтом вперед" (Big Endian), а ваш процессор x86 хранит их "младшим вперед" (Little Endian). Число `0x1234` в памяти лежит как `34 12`. Если прочитать его как есть, будет `0x3412` (13330 вместо 4660). Нужно перевернуть.
Пример для 16 бит
```C
unsigned short val = 0x3412; // То, что мы прочитали
unsigned short flipped = (val << 8) | (val >> 8);

// 1. val << 8:  0x3412 -> 0x1200 (сдвинули младший байт влево)
// 2. val >> 8:  0x3412 -> 0x0034 (сдвинули старший байт вправо)
// 3. OR:        0x1200 | 0x0034 = 0x1234.
```

5) **Инверсия цветов**
```C
unsigned int pixel = 0x00FF00; // Зеленый (R=0, G=255, B=0)
unsigned int negative = ~pixel; 

// ~0000 0000 1111 1111 0000 0000
//  1111 1111 0000 0000 1111 1111
// Result: 0xFF00FF (Фиолетовый: Red + Blue).
```
