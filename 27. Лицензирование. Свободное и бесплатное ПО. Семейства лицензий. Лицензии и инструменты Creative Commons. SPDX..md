***Лицензия*** - юридическое соглашение, определяющее правила использования, распространения и изменение ПО или кода, защищенного авторским правом.

***Лицензионный договор*** - документ, которым подкрепляется лицензия.

---
### Свободное и открытое ПО
Свободное ПО это компиляция открытого ПО и бесплатного ПО.

**Открытое ПО**
Исходный код находится в открытом доступе, но при этом его можно монетизировать различными способами.

**Бесплатное ПО**
Не нужно платить деньги за использование, при этом ПО может не находиться в открытом доступе, а лицензия может запрещать распространение

То есть в свободном ПО код должен быть открыт, но при этом за него не должны просить деньги.

Степень свободы ПО определяется аспектами:
- нужно ли платить
- открыт ли исходный код
- можно ли зарабатывать используя это по
- можно ли распространять и модифицировать

### Семейства лицензий GNU GPL (LGPL), BSD(MIT)

**GNU GPL (GNU General Public License)**
- Исходный код должен быть открытым
- ПО можно запускать, модифицировать, распространять
- если исходный код под GPL то весь проект должен использовать GPL
- яркий представитель движения copyleft
Примеры: Git, Linux Kernel

**LGPL (Lesser General Public License)**
- облегченная версия GPL для библиотек
- Софт может линковаться с библиотекой под LGPL, но исходники программы открывать не обязан
- Но если изменить библиотеку то, изменения должны быть открыты
Примеры: gllibc

**MIT**
Позволяет:
- использовать
- копировать
- изменять
- распространять
ПО без ограничений при условии, что:
- ПО и производные от него распространяются с текстом лицензии и указанием авторства
- не дается никаких гарантий (качества и тд)
- авторы не несут никакой ответственности по искам за ущерб или другим основаниям
Примеры: React, Node.js

**BSD (Berkeley Software Distribution)**
Очень похожа на MIT, но запрещает использовать имя автора или название организации для рекламы своего продукта.
Примеры: Nginx, Go, FreeBSD, OpenSSH

**Apache License 2.0**
Позволяет использовать, распространять, изменять в любых целях, при условии, что:
- Сохраняются все уведомления об авторстве, патентах, товарных знаках
- Включается текст лицензии
- Никаких гарантий не предоставляется, авторы никакой ответственности не несут
- При подаче патентного иска лицензия обнуляется
Примеры: Android, TensorFlow, Kubernetes

### Лицензии и инструменты Creative Commons. SPDX.
***Creative Commons*** - организация, написавшая семейство лицензий, которые автор может комбинировать под свои нужды.
Для ПО обычно не используются
- **CC BY** - необходимо указать автора
- **CC NC** - использование только в некоммерческих целях
- **CC ND** - нельзя изменять
- **CC SA** - производные должны идти с той же лицензией
- **CC0** - отказ от исключительного права (продукт становится общественным достоянием)

### SPDX
**Software Package Data Exchange** - международный открытый стандарт для описания состава ПО, лицензий и авторских прав в машиночитаемом формате.

Ключевые аспекты:
- **Стандартизация идентификаторов**. Вводит реестр уникальных коротких имен для лицензий (MIT, GPL-3.0-only и тд), устраняя путаницу в наименованиях
- **Формализация выражений**. Определяет синтаксис для описания сложных лицензированных условий (операторы `AND`, `OR`, `WITH`)
- **Автоматизация**. Позволяет инструментам разработки сканировать код и формировать отчет о составе ПО и его правовом статусе.

Используется в заголовках исходных файлов в виде комментария
	``// SPDX-License-Identifier: MIT``
