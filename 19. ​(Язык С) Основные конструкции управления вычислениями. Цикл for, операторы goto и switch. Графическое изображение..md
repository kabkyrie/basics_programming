### Основные конструкции управления
- Оператор безусловного перехода `goto`
	`goto` - это оператор безусловного перехода. Он заставляет программу мгновенно прыгнуть к строчке кода, помеченной определенной **меткой**

- Условный оператор - `if`
	Здесь стоит упомянуть про тернарный оператор, это оператор, и он возвращает какое-то значение.

- Цикл с предусловием `for` `while`
	Цикл for является синтаксическим сахаром в си, т.к. его реализация никак не отличается от while. Проверка условия выполняется перед итерацией, то есть может быть ноль итераций.

- Цикл с постусловием `do {} while ()`
	Также является синтаксическим сахаром, вообще все циклы это синтаксический сахар - смесь goto и if (с этим нужно поосторожнее, а то в ассемблер скатимся). Проверка осуществляется после итерации, то есть будет хоть одна итерация.

- Оператор выбора - `switch() { case X: … ; break; default: …. }`

---
### Цикл for, операторы goto и switch.

`for(s1, c s2)`:
- В начале цикла выполняет инструкции `s1`
- Перед каждой итерацией проверят, что `c` = `True`
- После каждой итерации выполняется инструкция `s2`

`goto(LABEL)`
- Делает безусловный переход (прыжок) на метку `LABEL`
- Метки объявляются как `LABEL:`
- После `goto` начинают исполнятся все инструкции после метки
- При этом возврата не происходит

`switch(expr) { case VAL: }`
- Делает прыжок на case если `expr` == `VAL`
- В качестве `VAL` могут быть только значения, полноценных выражений быть не может
- Чтобы выйти из тела `switch` используется `break`
- Метки сравниваются сверху вниз, если ни одно значение не совпало, делается прыжок на `default:`

### Блок схема
***Блок схема*** - представление программы в виде геометрических фигур, каждая из которых обозначает некоторую языковую конструкцию (цикл или ветвление), а также стрелок/линий, обозначающих переходы между частями программы.

Основные конструкции и обозначения:
- Скругленный прямоугольник - начало и конец программы
- Прямоугольник - действие, меняющее состояние программы
- Ромб - `if - then - else`
- Параллелограмм - операции ввода/вывода

### Граф потока управления
***Граф потока управления*** (CFG - control flow graph) - граф, вершинами которого являются базовые блоки, а ребрами - переходы управления.

***Базовый блок*** - (наибольшие) последовательности инструкций, выполняемых без перехода управления.

Переходом управления принято считать:
- Вызов функции
- Ветвление
- Цикл
- Прыжок на метку

```C
// начало базового блока
int a = 1;
int b = a + 1;
// конец базового блока

if (b > a)
{
	// начало базового блока
	a = b;
	// конец базового блока
}
else
{
	// начало базового блока
	int c = b;
	b = a;
	// конец базового блока
	
	printf("%d", b); // управление куда-то передалось
	
	// начало базового блока
	b = c;
	// конец базового блока
}

a = 0
```
